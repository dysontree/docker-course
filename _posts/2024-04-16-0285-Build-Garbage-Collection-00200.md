Слишком большой и старый сборочный кэш занимает место и может замедлять систему. Обычно мы включали в пайплайне некоторый скрипт, который мог использовать `docker build` с флагом `--no-cache`, также почистить кэш билдера и удалить старые образы.

```
docker builder prune
```

```
docker image rm $(docker image list --filter "until=7h" -q)
```

```
docker image build -t custom-nginx . --no-cache
```

Для удаления старых файлов в кэше можно запускать команду `docker builder prune` по cron-расписанию, или вызывать её внутри специальной очищающей job в нашей CIСD-системе, или использовать специальный образ для процедуры очистки.
